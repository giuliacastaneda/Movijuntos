## Relatório de testes de software

**1. Pontos Positivos Identificados**

Durante a execução dos testes, foram observados diversos aspectos positivos que contribuem diretamente para a eficiência e a usabilidade da aplicação:

<li>Navegação Intuitiva: A estrutura das páginas, as rotas e a organização dos elementos permitem que o usuário compreenda rapidamente como realizar tarefas como buscar locais, visualizar detalhes e registrar avaliações.
<li> Cumprimento do requisito RNF-005 (máx. 4 cliques): Os fluxos principais como cadastro, login, busca de locais, avaliação e cadastro de locais, podem ser concluídos dentro do limite de até 4 cliques, garantindo eficiência no uso e reduzindo a carga cognitiva do usuário.
<li> Carregamento Rápido dos Mapas: Os testes de desempenho mostraram que o mapa carrega dentro do limite estabelecido no requisito RNF-009 (≤ 15s). A média obtida demonstra boa otimização.
<li> Interface organizada e acessível: Os elementos possuem contraste adequado, textos de boa leitura e organização consistente com boas práticas de usabilidade. Isso contribui para que o sistema seja simples e agradável de usar.

**2. Fragilidades e Pontos Fracos Identificados**

Apesar dos aspectos positivos, algumas fragilidades foram detectadas:

<li>  Responsividade do mapa: Em alguns testes, ao redimensionar a janela, o mapa perdia a configuração ou deixava de renderizar corretamente. Isso prejudica usuários em dispositivos diferentes (notebook, tablet, celular). O mapa sumindo gera insegurança e sensação de instabilidade.
<li> Campos sem validação completa: Em alguns formulários (ex.: cadastro de locais), determinados campos não exibem feedback visual ou mensagens de erro adequadas quando preenchidos de forma incorreta. Campos sem validação clara dificultam o preenchimento.
<li>  Layout inconsistente em algumas páginas: Durante os testes de cliques e fluxo, observou-se que páginas sem container (página de mapa em tela cheia) afetaram o alinhamento e centralização de outras telas que dependem do layout padrão. Diferenças visuais entre telas geram estranhamento.
<li>  Dependência de recarga manual em alguns momentos: Em situações específicas, o mapa só carregou corretamente após atualização manual da página. Isso impacta negativamente a experiência do usuário.

**3. Estratégias de Correção e Melhorias**

Com base nos testes realizados, o grupo propõe as seguintes ações:<br>
→ Ajustes no código:
Reconfigurar o container e classes CSS para que apenas páginas específicas utilizem largura total, sem afetar o layout global;
Ajustar o script de renderização do Leaflet (ou Google Maps), especialmente o uso de map.invalidateSize() para corrigir renderizações após resize.

→ Melhorias na interface:
Adicionar mensagens claras de erro/validação nos formulários; 
Padronizar margens, espaçamentos e alinhamentos do layout.

→ Melhoria na experiência de avaliação:
Destacar visualmente critérios obrigatórios; 
Criar feedbacks imediatos ao selecionar “Sim” ou “Não”.

→ Testes adicionais: 
Realizar testes em múltiplos tamanhos de tela e dispositivos; 
Repetir o teste de carregamento periodicamente para garantir estabilidade.

**4. Conclusão e Ganhos Obtidos**

A partir dos testes, foi possível:
<br>
→  Validar que os requisitos principais foram atendidos. <br>
→  Identificar pontos cruciais de usabilidade, como responsividade e feedback ao usuário.<br>
→  Propor melhorias que aumentam a confiabilidade da solução, garantindo que o MoviJuntos seja mais acessível, estável e agradável de usar.<br>
→  Aumentar a maturidade do projeto, aproximando-o de uma aplicação real, com preocupações de desempenho, padronização e experiência do usuário.
